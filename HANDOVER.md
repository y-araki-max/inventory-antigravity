# プロジェクト引き継ぎメモ（内容）

## プロジェクト概要
- 在庫管理アプリ（入出庫管理）
- 現場のアナログ業務を、業務フローを極力変えずにデジタル化する目的
- 入力スピード・実用性最優先

## 技術構成
- フロントエンド：React + Vite + Tailwind
- バックエンド：Node（backend フォルダあり）
- リポジトリ構成：

  inventory-antigravity/
    ├ backend
    ├ inventory-app（frontend）
    └ 各種設定ファイル

## 利用状況・制約条件
- 入出庫伝票：1日5〜20枚程度
- 同時入力者：1人
- 使用端末：iPad（共有）
- 時間勝負の業務のため、大きな設計変更は避ける
- 「今のやり方を大きく変えない」ことが最重要

## 現在の実装状況（重要）

### InputDataList.jsx
- モバイル最適化済み
- テーブルレイアウト再設計
- 列幅調整：
  - Type（w-10）
  - ID / Staff（w-16）
  - Checks（w-8）
- テキストサイズ：text-xs（約12px）
- ヘッダー文言を短縮・日本語統一
  - 日付 / 種別（出 or 入）
  - BOSS / Staff → ID / 担当者
  - 社員 → 社販
  - BOSS Check → BOSS
  - 削除 → 削除

### Aggregation.jsx
- 集計結果の並び順を完全固定
- js/data.js の PRODUCTS 配列順を厳守
- 並び順例：
  - Large Bottle → Small Bottle → Premium …
- 管理・確認用途の一貫性確保が目的

### UI調整（出庫画面）
- カテゴリボタン配色：
  - 通常時：bg-blue-50
  - 選択時：bg-blue-600
- 視認性・操作性向上のため

## 用語統一ルール（最重要・ブレ厳禁）
以下は UI・コード・表示文言すべてで統一すること。
ズレはバグ扱い。

- いい → アミノ酸
- 有細胞子 → 有胞子
- エネルギー → エナジー
- 受け取り入力 / 受取 → 出庫入力 / 出庫
- ボス / ぼす → BOSS
- BOSS / Staff → ID / 担当者
- 社員 → 社販
- BOSS Check → BOSS
- 削除 → 削除（変更なし）

※「他商品」という表記が混入する回帰バグが過去に発生しているため注意

## 作業方針・注意点
- 大規模リファクタリングは禁止
- UIや用語は「現場が迷わない」ことを最優先
- AI Agent が途中で停止することがあるため、
  1タスクずつ確実に進める
